    Validating Forms:

  • There are several different methods for client-side validation, ranging anywhere from simply requiring user input to preventing form submission until all inputs are filled and match the requested input-pattern.
  • When requiring user input for a textbox, the required attribute must be used. For example:
    • <input type="text" placeholder="Enter some text" required>
    This will help prevent form submission until this textbox is filled, but will not ultimately prevent hackers from submitting a form. Other measures should be taken to perform such a prevention.
  • Further measures that can be taken for form-submission security are:
    1. Constraining the length of your entries (i.e., "minlength='6' maxlength='6'")
    2. Constraining the values of your entries (i.e., value="1" min="1" max="10")
    3. The Constraint Validation API (i.e., 'validationMessage', 'validity', 'checkValidity()', etc.)
    4. Implementing a customized error message (i.e.,
       if (email.validity.typeMismatch) {
        email.setCustomValidity("I am expecting an e-mail address!");
       })
    5. ...and so on.
  • When these actions are taken, we effectively prevent any hackers from sending dangerous malware to our web server, thus protecting ourselves and any visitors who wish to view or participate in our website.

    Using Fetch:

  • Fetch serves as a way to call data from an API platform or pipeline, serving as the replacement for XMLHTTPRequest. While XMLHTTPRequest is still available for use, Fetch API methods are much simpler to use and provide more flexibility.
  • A simple Fetch statement is written as follows:

     fetch('http://example.com/movies.json')
     .then(response => response.json())
     .then(data => console.log(data));

    This method simply fetches data from an existing API object, returns a response object in JSON format, and then displays the resulting data to the console.
  • Fetch methods can also upload files through the method PUT. Basically, the user needs to provide form data, a username, and an avatar to upload files. Once this information is provided, the upload method will push the file as well as display a success or failure message, dependent on the file being sent or not.
  • For unsupporting browsers, Fetch has a method called Fetch Polyfill, which basically fulfills the same purposes as standard Fetch methods. According to romainmenke and JakeChampion, "[Polyfill is a project] that implements a subset of the standard Fetch specification, enough to make fetch a viable replacement for most uses of XMLHttpRequest in traditional web applications."
